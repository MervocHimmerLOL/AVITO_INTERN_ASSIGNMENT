## TC-001: Cоздание объявления с валидной структурой.
**Шаги:**
1. Отправить POST-запрос на /api/1/item с телом, содержащим валидные данные объявления, например:
{
  "sellerID": 123,
  "name": "Тестовое объявление",
  "price": 5000,
  "statistics": {
    "likes": 0,
    "viewCount": 0,
    "contacts": 0
  }
}

**Ожидаемый результат:**
- 200 OK на запрос
- Ответ валидного типа, пример: {
    "status": "Сохранили объявление - 81d628b7-a012-49b9-b16d-918c271db091"
}
- Созданное объявление имеет валидную структуру
---

## TC-002: Получение объявления по существующему идентификатору
**Предусловие**
1. Создать валидное объявление

**Шаги:**
1. Отправить GET-запрос на /api/1/item/{id} с корректным идентификатором

**Ожидаемый результат:**
- 200 OK
- Структура соответствует схеме (см. TC-001)

---

## TC-003: Получение всех объявлений продавца по его ID

**Предусловия:**

Известен sellerID, у которого есть хотя бы одно объявление

**Шаги:**
1. Отправить GET-запрос на /api/1/{sellerID}/item

**Ожидаемый результат:**
- 200 OK
- Тело ответа — JSON-массив
- Массив содержит хотя бы один элемент с ожидаемой структурой (см. TC-001)
- Все элементы в массиве соответствуют указанному sellerID

---

## TC-004: Получение статистики объявления по ID 
**Предусловия:**

— Объявление создано (известен его id)

**Шаги:**
1. Отправить GET-запрос на /api/1/statistic/{id}
2. Отправить GET-запрос на /api/2/statistic/{id}

**Ожидаемый результат:**
- Оба запроса возвращают 200 OK
- Тело ответа — JSON-массив из одного объекта: 
[
  {
    "likes": 0,
    "viewCount": 0,
    "contacts": 0
  }
]
---

## TC-005: Создание объявления без обязательного поля sellerID
**Цель:** Проверить валидацию входных данных.

**Шаги:**
1. Отправить POST-запрос на /api/1/item с телом, в котором отсутствует sellerID, например: 
{
  "name": "Некорректное объявление",
  "price": 5000,
  "statistics": {
    "likes": 0,
    "viewCount": 0,
    "contacts": 0
  }
}

**Ожидаемый результат:**
- 400 Bad Request
- Тело ответа
{
    "result": {
        "message": "поле sellerID обязательно",
        "messages": {}
    },
    "status": "400"
}
